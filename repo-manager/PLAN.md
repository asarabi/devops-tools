# Repo Manager - Plan

## 목적

다중 Gerrit 인스턴스 및 GitHub 레포지토리를 통합 관리하는 웹 대시보드를 제공한다.
레포지토리 현황, 브랜치 정보, 권한 상속 구조를 한 곳에서 파악할 수 있게 한다.

## 현재 상태

- [x] FastAPI + SQLite 기반 백엔드
- [x] Gerrit REST API / SSH Sync 구현
- [x] GitHub API Sync 구현
- [x] 웹 대시보드 UI (Bootstrap 5 다크 테마)
- [x] 레포 목록, 검색, 필터링
- [x] 레포 상세 (브랜치, 권한) 모달
- [x] Gerrit 권한 상속 트리 시각화
- [x] 주기적 자동 Sync (APScheduler)
- [x] 수동 Sync 트리거
- [x] Docker Compose 배포 (포트 8081)
- [x] README 작성

## 향후 계획

### 단기

- [ ] Gerrit/GitHub 실제 인스턴스 연동 테스트
- [ ] Sync 에러 핸들링 강화
  - 네트워크 타임아웃, 인증 실패 시 UI에 에러 상태 표시
- [ ] Sync 이력 관리
  - 언제, 어떤 소스가, 몇 개의 레포를 동기화했는지 이력 저장
- [ ] 페이지네이션
  - 레포가 수천 개일 때 대비

### 중기

- [ ] 레포 비교 기능
  - 두 Gerrit 인스턴스 간 레포/브랜치 diff
  - 동일 이름 레포의 브랜치 차이 시각화
- [ ] 권한 분석 리포트
  - 특정 그룹이 접근 가능한 전체 레포 목록
  - 과도한 권한 부여 경고
- [ ] GitHub 추가 정보
  - PR 현황, 최근 활동, 보호된 브랜치 정보
- [ ] 사용자 인증
  - 대시보드 접근 시 로그인 필요

### 장기

- [ ] Webhook 기반 실시간 Sync
  - Gerrit/GitHub 이벤트 수신 시 해당 레포만 업데이트
- [ ] 알림 기능
  - 새 레포 생성, 브랜치 삭제, 권한 변경 시 Slack/Email 알림
- [ ] 다중 사용자 + 역할 기반 접근 제어
- [ ] PostgreSQL 마이그레이션 (대규모 데이터 시)
